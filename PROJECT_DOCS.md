# 🚀 Recommendation Serving & Analytics Platform: Technical Specification

## 1. Executive Summary
The Recommendation Serving & Analytics Platform is an enterprise-grade system designed to solve the "latency vs. scale" trade-off in recommendation systems. Traditional methods of querying large datasets (via AWS Athena or SQL) often result in multi-second latencies, which are unacceptable for real-time applications. 

This platform utilizes an **In-Memory Cache-Aside** architecture, where the serving layer maintains a pre-processed mirror of the S3 data source in RAM, enabling sub-10ms response times while providing a rich analytical dashboard for model governance.

---

## 2. System Architecture

### 2.1 High-Level Design
The system is split into three distinct tiers:
1.  **Storage Tier (AWS S3)**: Acts as the immutable source of truth. Data is stored as CSV or Parquet files generated by a SageMaker pipeline.
2.  **Serving Tier (FastAPI)**: A high-performance Python backend that loads the entire recommendation set into a specialized dictionary structure upon boot.
3.  **Presentation Tier (React + Vite)**: A modular frontend using Nivo for complex data visualizations and Tailwind-inspired CSS for a "glassmorphism" UI.

### 2.2 The "Load-to-Memory" Strategy
Instead of querying a database per request, the backend performs a **Bulk Load** during the `lifespan` startup event:
- **Time Complexity**: $O(N)$ during startup (where $N$ is total recommendations).
- **Space Complexity**: $O(N)$ in RAM.
- **Lookup Complexity**: $O(1)$ for customer retrieval.

## 3. Backend Deep Dive

### 3.1 Data Loading Engine (`data_loader.py`)
The `data_loader` is the core of the system. It handles the transformation of raw S3 files into optimized memory structures.

#### A. The Lookup Structure
The data is stored in a `defaultdict(list)` called `_lookup`:
```python
_lookup = {
    "CUST_123": [
        {"product_id": "P1", "score": 0.95, "lift": 4.2, ...},
        {"product_id": "P2", "score": 0.88, "lift": 3.1, ...},
    ],
    ...
}
```
This structure allows the `/recommendations/{id}` endpoint to simply perform a dictionary key access, avoiding all compute at request time.

#### B. Statistics Computation (`_compute_stats`)
To keep the `/stats` endpoint instantaneous, aggregates are computed once during loading:
- **Quality Mix**: Scans the `trigger_product` column. If the value is `fallback`, it's categorized as baseline; otherwise, it's "Association-based".
- **Model Health**: Injects Silhouette scores (representing how well-defined the clusters are) and a distribution of customer counts per cluster.
- **Category Performance**: Uses Pandas `groupby` on `l2_category` to calculate the mean score and lift per category.
- **Distributions**: Uses `numpy.histogram` to create frequency buckets for Lift and Score, which are then mapped to string labels (e.g., "1.00–1.50") for the frontend.

### 3.2 API Design & Endpoints
The API is built with FastAPI and Pydantic for automatic validation and OpenAPI documentation.

| Endpoint | Method | Purpose | Complexity |
| :--- | :--- | :--- | :--- |
| `/recommendations/{id}` | `GET` | Returns top $N$ recs for a user | $O(1)$ |
| `/stats` | `GET` | Returns pre-computed global analytics | $O(1)$ |
| `/data/refresh` | `POST` | Clears memory and re-loads from S3 | $O(N)$ |
| `/pipeline/run` | `POST` | Triggers SageMaker Execution | $O(1)$ |
| `/pipeline/status` | `GET` | Polls SageMaker for current state | $O(1)$ |

## 4. Frontend Deep Dive

### 4.1 State & Data Synchronization
The dashboard implements a **Polling Mechanism** (`setInterval` every 30s) to ensure that if the data is refreshed on the backend, the charts update automatically without a page reload.

### 4.2 Visualization Logic
The platform uses `@nivo` for SVG-based charts:
- **ResponsiveBar**: Used for Model Health and Category Performance. Configured with a custom theme to match the dark/muted aesthetic.
- **ResponsivePie**: Used for Quality Mix. Implements `innerRadius={0.6}` to create a donut chart.
- **DistributionHistogram**: A custom wrapper around a Bar chart that visualizes the density of model confidence.

### 4.3 Segment Filtering Logic
The "Segment" filter allows users to drill down into specific customer groups. 
- **Logic**: The filter checks if the `selectedSegment` (e.g., `Electronics_General`) starts with the product's `category` (e.g., `Electronics`).
- **Impact**: This dynamically updates the "Top Recommended Products" table, allowing business users to see what products are driving lift within specific market segments.

## 5. Model Governance Metrics

### 5.1 Quality Mix (Association vs Fallback)
- **Association**: Recommendations generated based on strong co-occurrence or similarity (High Signal).
- **Fallback**: Generic recommendations used when no strong signal exists for a customer (Baseline).
- **Goal**: A higher Association % indicates a more personalized and effective model.

### 5.2 Model Health (Silhouette Score)
The system tracks the **Average Silhouette Coefficient**:
- **Value $\approx 1$**: Clusters are well-separated and dense.
- **Value $\approx 0$**: Overlapping clusters.
- **Value $< 0$**: Samples assigned to the wrong cluster.

## 6. Operational Guide

### 6.1 Environment Configuration
| Variable | Default | Description |
| :--- | :--- | :--- |
| `MOCK_MODE` | `true` | If true, generates 10,000 synthetic rows for testing. |
| `S3_BUCKET` | `ipre-prod-poc` | AWS Bucket name for data. |
| `S3_KEY` | `recs.csv` | Path to the CSV/Parquet file. |
| `SAGEMAKER_PIPELINE_NAME`| `mock-pipeline`| The pipeline ID in AWS SageMaker. |

### 6.2 Deployment Workflow
1. **Data Update**: SageMaker pipeline finishes $\rightarrow$ Writes file to S3.
2. **Sync**: Administrator calls `POST /data/refresh`.
3. **Verify**: Dashboard updates via polling $\rightarrow$ KPIs reflect new data.
4. **Monitor**: Check "Quality Mix" to ensure the new model didn't increase fallback rates.

## 7. Troubleshooting

- **White Screen on Dashboard**: Usually caused by missing data in the S3 file. Fixed via optional chaining (`?.`) in the React components.
- **Slow Startup**: Occurs if the S3 file is multi-gigabyte. Recommendation: Switch from `.csv` to `.parquet` for faster loading and smaller memory footprint.
- **Pipeline Status "UNKNOWN"**: Occurs if the `SAGEMAKER_PIPELINE_NAME` is incorrect or AWS credentials have expired.
